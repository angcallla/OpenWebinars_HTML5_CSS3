+Etiquetas: Accesibilidad

En nuestro país, por ley, nuestas páginas deben cumplir ciertos requisitos de accesibilidad

Contruir nuestra web teniendo presente que cualquier app o persona puede acceder a ella, incluso por aquellos que presentan una discapacidad

Algunas reglas generales

  • utilizar etiquetas semánticas en vez de div, para que los dispositivos sepan lo que hay dentro
  • usar el atributo "for" para los labels. Esto es la asociación que veniamos haciendo del atributo "for" del label y el atributo "name" del input
  • Añadir atributo "alt" en las imágenes
  • Especificar el idioma de la página
  • Utilizar las cabeceras correctamente (h1, h2, ...)

Atributos ARIA (Novedad en HTML5)

Los atributos ARIA (Accesible Rich Internet Application), son atributos que se añaden a las etiquetas para que estas puedan ser mejor entendidas por las personas con alguna discapacidad.

Algunos de los atributos más importantes:

  • role: para indicar el rol que posee mi etiqueta. Por ejemplo, si es una alerta (alert), si es un documento (document), etc.
  • aria-owns: para expresar la jerarquía de una etiqueta
  • aria-hidden: para decir que ese campo está oculto
  • aria-live: para decir que es una zona de mi web que se va a ir actualizando constantemente
  • aria-check: que el control del formulario está seleccionado
  • aria-label: que es una etiqueta
  • aria-disabled: que es un campo que está desabilitado
  • aria-valuenow/max/min: para indicar en un campo de rango el valor actual, máximo o mínimo.
